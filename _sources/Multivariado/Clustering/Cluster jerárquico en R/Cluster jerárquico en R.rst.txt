Cluster jer√°rquico:
~~~~~~~~~~~~~~~~~~~

Usaremos la funci√≥n ``hclust()`` de la librer√≠a ``stat``.

En esta funci√≥n se debe pasar el vector con las distancias calculadas
con la funci√≥n ``dist()``.

Para realizar el agrupamiento la funci√≥n ``hclust()`` tiene varios
m√©todos: ``"ward.D"`` , ``"ward.D2"``, ``"single"``, ``"complete"``,
``"average"``, ``"mcquitty"``, ``"median"`` o ``"centroid"``.

-  ``"ward.D"`` y ``"ward.D2"``: minimiza la varianza total dentro del
   cluster. En cada paso, une el par de cluster con una distancia m√≠nima
   entre clusters. ``"ward.D2"`` eleva al cuadrado las distancias
   calculadas con ``dist()``.

-  ``"single"``: a diferencia del anterior, lo hace con el valor m√≠nimo.
   Tiende a producir grupos largo. Es el m√©todo de la distancia m√≠nima.

-  ``"complete"``: la distancia entre dos grupos se define como el valor
   m√°ximo de todas las distancias por pares entre los elementos del
   grupo 1 y los elementos del grupo 2. Tiene a producir cluster m√°s
   compactos. Es el m√©todo de la distancia m√°xima.

-  ``"average"``: la distancia entre dos clusters se define como la
   distancia promedio entre los elementos del cluster 1 y los elementos
   del cluster 2. Es el m√©todo del promedio.

-  ``"mcquitty"``: promedio ponderado.

-  ``"median"``: mediana.

-  ``"centroid"``: usa las distancias entre el centroide 1 y el
   centroide 2.

.. code:: r

    hc <- hclust(d = dist, method = "ward.D2")
    print(hc)

Dendograma:
~~~~~~~~~~~

Visualizaremos el dendograma con la funci√≥n ``fviz_dend()`` de la
librer√≠a ``factoextra``.

Los principales argumentos de esta funci√≥n son:

-  ``x``: el objeto creado con ``hclust()``.

-  ``k``: cantidad de clusters. Primero no especificaremos este valor.

-  ``h``: para cortar el dendograma por altura. Primero no
   especificaremos este valor.

.. code:: r

    library("factoextra")

Distancia cofen√©tica (cophenetic distances):
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Para comprobar si las distancias del √°rbol reflejan las distancias
originales. Se calcula la correlaci√≥n entre las distancias cofen√©ticas y
los datos de las distancias originales generadas por la funci√≥n
``dist()``.

Cuanto m√°s cerca est√© el valor del coeficiente de correlaci√≥n a 1, con
mayor precisi√≥n la soluci√≥n de agrupaci√≥n reflejar√° sus datos.

La distancia cofen√©tica del √°rbol se calcula con la funci√≥n
``cophenetic()``.

.. code:: r

    coph <- cophenetic(hc)

.. code:: r

    cor(dist, coph)

    0.802735921415624


Corte del dendograma:
~~~~~~~~~~~~~~~~~~~~~

El √°rbol se puede cortar a una altura determinada o por cantidad de
clusters.

La funci√≥n de la base de R, ``cutree()`` permite cortar el √°rbol
generado por ``hclust()``\ por cantidad de clusters, ``k``, o por
altura, ``h``. Devuelve un vector que contiene el n√∫mero de cluster de
cada observaci√≥n.

:math:`k = 3`

.. code:: r

    grp <- cutree(hc, k = 3)
    print(head(grp))           # vector con los n√∫meros del cluster de cada observaci√≥n.

Otra forma:
~~~~~~~~~~~

Tenemos dos funciones del paquete ``cluster`` que calculan el
agrupamiento aglomerativo y divisivo.

-  ``agnes()``: aglomerativo.

-  ``diana()``: Divisivo.

Con estas funciones no se necesitan ejecutar ``scale()``, ``dist()`` y
``hclust()`` por separado.

.. code:: r

    library("cluster")


.. parsed-literal::

    Warning message:
    "package 'cluster' was built under R version 4.1.3"
    

.. code:: r

    agnes <- agnes(x = df,                       # Conjunto de datos original.
                   stand = TRUE,                  # Escalamiento de las variables.
                        metric = "euclidean",     # M√©todos para las distancias.
                        method = "ward")         # M√©todo para el agrupamiento.
                    
    
    diana <- diana(x = df,
                   stand = TRUE, 
                   metric = "euclidean") 

.. code:: r

    fviz_dend(agnes, k = 3)
